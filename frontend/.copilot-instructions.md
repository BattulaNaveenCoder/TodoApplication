# Copilot instructions — Frontend (React / TypeScript)

## TypeScript
- Strict mode enabled in tsconfig — no implicit any.
- All props have explicit interface definitions.
- Never use type any — use unknown and narrow it.
- type for unions, interface for object shapes.

## React components
- Functional components only — no class components.
- One component per file.
- File name matches component name (PascalCase).
- Props interface named <ComponentName>Props.
- No inline arrow functions in JSX that cause re-renders.
  Extract to useCallback or module level.

## State management
- useState for local UI state only.
- useReducer when state has multiple sub-values or transitions.
- No prop drilling beyond 2 levels — lift state or use context.
- API state (loading, error, data) managed via custom hook.

## API integration
- All API calls in /frontend/src/api/
- One file per resource: todosApi.ts, categoriesApi.ts
- Axios with a shared instance — baseURL from VITE_API_URL env var.
- All API functions async and return typed responses.
- Error handling at hook level — components receive clean state only.

## Custom hooks
- Hooks in /frontend/src/hooks/
- Named: useTodos, useCategories etc.
- Each hook encapsulates: fetch on mount, loading, error, mutations.
- Hooks return domain types — never raw API responses.

## Folder structure
```
src/
  api/          API call functions
  components/   Reusable UI components
  hooks/        Custom React hooks
  pages/        Page-level components
  types/        TypeScript interfaces and types
  utils/        Pure helper functions
```

## Testing
- Vitest + React Testing Library.
- Test behaviour, not implementation.
- No snapshot tests.
- Mock API calls with vi.mock() at module level.
- Test names: should <do something> when <condition>
- Every hook: test with renderHook().
- Every component: renders without crash, loading state,
  error state, data state.

## Performance
- React.memo only when profiler confirms re-render problem.
- Avoid useEffect for derived state — compute inline.
- No unguarded console.log in production code.

## Accessibility
- Interactive elements must have aria-label if no visible text.
- Form inputs must have associated label elements.
- Buttons must have explicit type attribute.
