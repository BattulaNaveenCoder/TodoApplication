# Copilot instructions — Backend (Python / FastAPI)

## Python standards
- Python 3.11+. Use match/case, TypeAlias, Self where appropriate.
- Full type hints on all functions and methods.
- Use Pydantic models or dataclasses — never plain dicts for data transfer.
- No **kwargs in public APIs.

## FastAPI
- All routers use APIRouter with prefix and tags.
- All endpoints are async def.
- DB session and services injected via Depends().
- Always set response_model on every endpoint.
- Explicit status codes: 200, 201, 204, 400, 404, 422, 500.
- No logic in endpoint functions — delegate to service immediately.

## SQLAlchemy
- Always use AsyncSession.
- Session injected via Depends — never created manually inside functions.
- All queries use select() — never session.query() (legacy API).
- Relationships loaded explicitly — no lazy loading in async context.
- Models in app/models/, schemas in app/schemas/.

## Alembic
- Never modify the database manually — always use Alembic.
- Schema change workflow:
  alembic revision --autogenerate -m "describe_change"
  alembic upgrade head
- Always review autogenerated migration before running it.

## Repository layer
- One repository class per model.
- Standard methods: get_by_id, get_all, create, update, delete.
- All methods async, accept AsyncSession as first param after self.
- Return model instances or None — never raise HTTP exceptions here.
- No business logic — data access only.

## Service layer
- One service class per feature domain.
- Receives repository via constructor injection.
- Contains all business logic and validation.
- Raises custom domain exceptions (not HTTPException).
- Handles None returns from repository — raises NotFoundError etc.

## Router layer
- One router file per resource: todos.py, categories.py
- Maps service exceptions to HTTP exceptions.
- Validates input via Pydantic schema — never raw request body.
- Returns Pydantic response schemas — never ORM models directly.

## Testing
- pytest-asyncio with asyncio_mode = "auto" in pytest.ini.
- Separate test DB connection — never use production DB.
- httpx.AsyncClient for endpoint integration tests.
- Mock repositories in service unit tests.
- Every repository method: unit test.
- Every service method: unit test with mocked repo.
- Every endpoint: integration test.

## Docstring format (Google style)
```python
def create_todo(self, data: TodoCreate) -> Todo:
    """Create a new todo item.

    Args:
        data: Validated todo creation schema.

    Returns:
        The newly created Todo ORM instance.

    Raises:
        DuplicateTitleError: If a todo with this title already exists.
    """
```
